Programación de aplicaciones con sockets

Interfaz de sockets

La estructura del kernel consta de tres capas: la capa de socket, la capa de protocolo y la capa de dispositivo.

La capa de socket proporciona la interfaz entre las subrutinas y las capas inferiores, la capa de protocolo contiene los módulos de protocolo utilizados para la comunicación y la capa de dispositivo contiene los controladores de dispositivo que controlan los dispositivos de red. Los protocolos y controladores se pueden cargar dinámicamente. La imagen siguiente ilustra la relación entre las capas.

Los procesos se comunican utilizando el modelo de cliente y servidor. En este modelo, un proceso de servidor, un punto final de una vía de acceso de comunicación bidireccional, escucha un socket. El proceso de cliente, el otro extremo de la vía de acceso de comunicación, se comunica con el proceso de servidor a través de otro socket. El proceso de cliente puede estar en otra máquina. El kernel mantiene conexiones internas y direcciona datos de cliente a servidor.

Dentro de la capa de sockets, la estructura de datos de sockets es el foco de la actividad. Las subrutinas de interfaz de llamada al sistema gestionan las actividades relacionadas con una subrutina, recopilando los parámetros de subrutina y convirtiendo los datos de programa al formato esperado por las subrutinas de segundo nivel.

La mayoría de los recursos de socket se implementan dentro de subrutinas de segundo nivel. Estas subrutinas de segundo nivel manipulan directamente las estructuras de datos de socket y gestionan la sincronización entre actividades asíncronas.
